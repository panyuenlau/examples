1. Create a test cluster: kind create cluster --name=test

2. Install NSM:
KCONF=/Users/darrelau/.kube/config nsm-nse/scripts/vl3/nsm_install_interdomain.sh

3. Install NSC, ICMP-responder network service and NSE:
kubectl apply -f examples/examples/icmp/k8s/example-responder

4. exec into the NSC pod and do a `ip a show nsm0`, `nsm0` is the network interface created to connect to the network service, example output:

37: nsm0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1450 qdisc mq state UNKNOWN group default qlen 1000
    link/ether 02:fe:fb:3e:f6:ec brd ff:ff:ff:ff:ff:ff
    inet 172.16.1.1/30 brd 172.16.1.3 scope global nsm0
       valid_lft forever preferred_lft forever
    inet6 fe80::fe:fbff:fe3e:f6ec/64 scope link 
       valid_lft forever preferred_lft forever

5. exec into the NSE pod and do a `vppctl sh int addr` to show the network interface address, example output:
root@ucnf-endpoint-5c58bb6577-2chv7:~# vppctl sh int addr
local0 (dn):
memif1/0 (up):
  L3 172.16.1.2/30

6. To test the connectivity between the NSC and NSE, exec into the NSC pod and do `ping 172.16.1.2`
